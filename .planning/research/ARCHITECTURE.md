## Data Flow

1. **Discovery**: Zeek captures HTTP metadata from SPAN/TAP port mirror вҶ’ Kafka topic (`raw-api-calls`)
2. **Feature Computation**: Flink consumes `raw-api-calls`, computes 20+ features per endpoint вҶ’ writes to Feast + Redis
3. **Classification**: XGBoost reads features from Redis вҶ’ classifies each endpoint вҶ’ outputs zombie probability + SHAP explanation
4. **Policy Evaluation**: OPA evaluates decommission policies against classification + metadata вҶ’ decision (approve/reject/escalate)
5. **Workflow**: GitHub Actions triggers decommission workflow вҶ’ opens GitHub issue (Stage 2) or runs canary (Stage 3)
6. **Enforcement**: Kyverno intercepts new API deployments вҶ’ verifies sunset-date and owner labels вҶ’ rejects non-compliant

## Build Order Dependencies

1. Phase 1: Infrastructure (K8s, Kafka, MinIO) вҖ” foundation for everything
2. Phase 2: Listen Layer (Zeek, Flink) вҖ” must have data flowing
3. Phase 3: Remember Layer (Feast, Redis) вҖ” must store features
4. Phase 4: Detect Layer (XGBoost, SHAP, MLflow) вҖ” must classify
5. Phase 5: Enforce Layer (OPA, Kyverno, Backstage) вҖ” must have policies
6. Phase 6: Act Layer (GH Actions, Flagger) вҖ” must have decommission workflow
7. Phase 7: Discovery Methods 2-5 вҖ” layered on top of core pipeline
8. Phase 8: Compliance & Governance вҖ” charter, audit trail, documentation
