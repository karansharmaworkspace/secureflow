# Compliance Documentation

Regulatory compliance and governance documentation for the SecureFlow platform. This directory maps the platform's features against banking and security standards to ensure regulatory adherence.

## Files

### compliance-checklist.md
Verification checklist covering all compliance requirements. Used during audits to confirm the platform meets regulatory obligations across OWASP, NIST, GDPR, PCI-DSS, RBI, and ISO standards.

### governance-charter.md
Defines the governance framework for the zombie API decommission process. Covers decision rights, escalation paths, approval chains, and oversight mechanisms for the three-stage decommission workflow.

### regulatory-mapping.md
Maps SecureFlow platform capabilities to specific regulatory requirements:
- OWASP API Top 10 (API2, API7, API9)
- NIST CSF 2.0 (Identify, Protect, Detect, Respond, Recover)
- GDPR Article 25 (data minimization, retention limits)
- PCI-DSS v4 (card data redaction, encryption at rest)
- RBI IT Framework (asset inventory, vulnerability SLAs)
- ISO 27001 A8.8 (full lifecycle management)
