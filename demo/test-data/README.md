# Test Data

Synthetic test data generated by the traffic simulator for ML model training and validation. Contains parquet files with endpoint feature datasets representing realistic banking API traffic patterns.

## Contents

### endpoint_features.parquet

**Source:** `demo/traffic-generator/simulate.py --generate-features`

**Specs:**
- ~1,13,836 rows × 30 columns
- File size: ~15-20 MB (parquet compressed)
- Format: Apache Parquet (columnar, compressed)

**Columns (30 total):**

| Column | Type | Description |
|--------|------|-------------|
| endpoint_key | string | `host\|method\|path` unique identifier |
| total_calls | int64 | Daily call volume |
| synthetic_calls | int64 | Bot/monitor calls |
| real_calls | int64 | Real user calls |
| days_since_last_real_call | int64 | Staleness in days |
| unique_user_agents | int64 | UA diversity count |
| unique_source_ips | int64 | IP diversity count |
| auth_ratio | float64 | 0.0–1.0 authenticated fraction |
| ratio_2xx | float64 | 0.0–1.0 success rate |
| ratio_4xx | float64 | 0.0–1.0 client error rate |
| ratio_5xx | float64 | 0.0–1.0 server error rate |
| response_size_mean | float64 | Avg response payload bytes |
| response_size_stddev | float64 | Response size variability |
| interarrival_mean_ms | float64 | Avg time between calls |
| interarrival_stddev_ms | float64 | Timing irregularity |
| unique_hours_of_day | int64 | Active hours spread |
| is_100pct_synthetic | int64 | 0/1 flag |
| is_zombie | int64 | **Label: 0=active, 1=zombie** |
| api_version, domain, host | string | Metadata |
| method, path | string | HTTP method + path |
| annual_cost_inr | float64 | Estimated yearly cost |

**ML Usage:**
- Training: `ml-pipeline/train.py` reads this file, splits 80/20 train/test
- Features: columns 2–17 (16 features), excludes `is_100pct_synthetic` (data leakage)
- Target: `is_zombie` (binary classification)
- ~50% zombie / 50% active class balance

**Regeneration:**
```bash
python demo/traffic-generator/simulate.py --generate-features
```
